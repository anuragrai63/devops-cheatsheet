üì¶ The Container

A runnable unit of software. Uses immutable Images.

Imperative:
kubectl run my-pod --image=nginx:latest kubectl set image deployment/dep c=img:tag
YAML Spec:
spec: containers: - name: app image: nginx:1.14.2 imagePullPolicy: IfNotPresent

üöÄ The Workload

Controllers (Deployments, StatefulSets) that manage Pod availability.

Imperative:
kubectl create deployment my-dep --replicas=3 kubectl get pods -o wide
YAML Spec:
apiVersion: apps/v1 kind: Deployment spec: replicas: 3 selector: matchLabels: ...

üç© Modern Pod Anatomy

A Pod is rarely just one container anymore. It's a collection of cooperative processes.

Configuration Blueprint

1. Init Container (YAML)
initContainers: - name: setup image: busybox command: ['sh', '-c', 'until nslookup db; do sleep 2; done']
2. Native Sidecar (YAML)
initContainers: - name: log-agent image: fluentd restartPolicy: Always # Magic Flag
3. Ephemeral Container (Imperative Only)
# Attach debug shell to running pod kubectl debug -it my-pod --image=busybox --target=main-container

üõ°Ô∏è Quality of Service (QoS) Classes

K8s assigns QoS based on requests/limits.

Guaranteed (The Gold Standard)

Request = Limit for everything.

resources: limits: memory: "200Mi" cpu: "700m" requests: memory: "200Mi" cpu: "700m"
Burstable
resources: requests: memory: "100Mi" # Limit > Request or No Limit
